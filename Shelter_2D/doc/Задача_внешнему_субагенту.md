# БЛОК 3 — КОРОТКОЕ СООБЩЕНИЕ ДЛЯ ЧАТА ВНЕШНЕМУ СУБАГЕНТУ

Основной агент даёт пользователю короткое сообщение для вставки в чат внешнему или локальному агенту.

Шаблон:

```text
Прочитай задачу внешнего субагента:

M:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_внешнему_субагенту.md

Эта задача подготовлена основным Codex.
Ты работаешь как управляемый внешний или локальный субагент через чат пользователя.
В начале работы и в отчёте укажи конкретного исполнителя: модель / среда / маршрут, например `GigaChat / VS Code / внешний через чат пользователя`.
Не принимай самостоятельные расширения задачи вне БЛОК 1.
Выполни активную задачу из БЛОК 1.
Соблюдай режим работы из задачи: edit только в указанном scope.
Читай и меняй только файлы, явно перечисленные в scope.
Всё, что не перечислено в scope, запрещено.
Git не используй вообще: ни для чтения, ни для проверки, ни для изменения.
Не трогай `.git`, `.obsidian`, Codex config, OpenCode config и VS Code config.
Важно: не бери активную задачу из `Задача_агенту.md` как источник требований. Источник задачи только БЛОК 1 файла `Задача_внешнему_субагенту.md`.
Важно: `Задача_внешнему_субагенту.md` разрешено менять только в `БЛОК 2 — РАСШИРЕННЫЙ ОТЧЁТ ВНЕШНЕГО СУБАГЕНТА`. `БЛОК 1`, `БЛОК 3` и остальные разделы этого файла не меняй.
Сохраняй Markdown как обычный UTF-8 текст с настоящими переносами строк. Не записывай файл одной строкой, не вставляй буквальные `\n`, не экранируй Windows-пути двойными слэшами.
Не очищай БЛОК 1 самостоятельно: после проверки это делает основной Codex.
Не удерживай сессию пустыми рассуждениями или псевдо-вызовами инструментов.
Если нужный инструмент недоступен, сразу напиши это в отчёте и остановись.

По завершении:
1. Обязательно верни блок `USER SUMMARY` — коротко для пользователя.
2. Обязательно запиши блок `AGENT EXTENDED REPORT` в БЛОК 2 файла задачи.
3. В поле `Agent` укажи конкретно: модель / среда / маршрут через чат пользователя.
4. В поле `Status` укажи итог: `успешно`, `частично` или `неуспешно`.
5. Если запись в `БЛОК 2` невозможна, верни расширенный отчёт в чат и честно укажи недоступный инструмент или permission.
6. Последними строками `USER SUMMARY` напиши один из маркеров:

```text
РАБОТА ЗАВЕРШЕНА
Итог: успешно
Контроль возвращён основному Codex.
```

или

```text
РАБОТА НЕ ЗАВЕРШЕНА
Итог: частично / неуспешно
Контроль возвращён основному Codex.
```
```

# ПРАВИЛА РАБОТЫ

Основной агент:

- пишет активную задачу в `БЛОК 1`;
- даёт пользователю короткое сообщение из `БЛОК 3`;
- после выполнения читает короткий отчёт пользователя и расширенный отчёт из `БЛОК 2` или из чата;
- возвращает в основной чат краткую выдержку как `SUBAGENT REPORT`;
- в `SUBAGENT REPORT` всегда указывает конкретного агента и то, что обращение шло через чат пользователя;
- проверяет результат и принимает финальное решение сам.

Внешний или локальный агент через чат пользователя:

- читает этот файл;
- выполняет только `БЛОК 1`;
- не выходит за scope;
- работает локально на ПК и не использует Git;
- даёт пользователю короткий отчёт;
- пишет расширенный отчёт в `БЛОК 2` или возвращает его текстом пользователю;
- не очищает `БЛОК 1` самостоятельно, если это не разрешено отдельной задачей;
- завершает ответ явным маркером `РАБОТА ЗАВЕРШЕНА` или `РАБОТА НЕ ЗАВЕРШЕНА`;
- сохраняет Markdown обычным UTF-8 текстом с настоящими переносами строк.

Если задача внешнего или локального агента через чат пользователя успешно завершена и отчёт принят, основной агент сначала проверяет, что расширенный отчёт записан в `БЛОК 2`, а затем очищает `БЛОК 1` до состояния "Активной задачи для внешнего субагента нет".

Если задача основного агента успешно выполнена, основной агент сначала записывает итоговый отчёт в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА` файла `Задача_агенту.md`, а затем очищает `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА` в `Задача_агенту.md`.

Если задача выполнена частично, неуспешно или требует решения пользователя, `БЛОК 1` не очищается, а причина указывается в `PROBLEMS`.
