# MANIFEST структуры шаблона

Template version: 0.2

Этот документ фиксирует обязательную структуру шаблона агентской системы.

Агент не должен начинать перенос с частного шаблона. Сначала читаются:

1. `README_как_переносить.md`
2. `MANIFEST_структуры.md`
3. `00_Паспорт_проекта.md`
4. `02_Карта_переменных.md`
5. `01_Правило_подгонки_под_новый_проект.md`
6. `03_Чеклист_проверки_переноса.md`

## Обязательные файлы

```text
Шаблон_агентской_системы\
├─ README_как_переносить.md
├─ MANIFEST_структуры.md
├─ 00_Паспорт_проекта.md
├─ 01_Правило_подгонки_под_новый_проект.md
├─ 02_Карта_переменных.md
├─ 03_Чеклист_проверки_переноса.md
├─ 04_Dry_run_пример_переноса.md
├─ Шаблоны_документов\
│  ├─ Старт.md
│  ├─ Задача_агенту.md
│  ├─ Список_правил.md
│  ├─ Правила_Git_и_Review.md
│  ├─ Правила_создания_документов.md
│  ├─ Правила_использования_субагентов.md
│  ├─ Настройка_прав_субагентов.md
│  └─ Задача_внешнему_субагенту.md
└─ Формы_отчётов\
   ├─ AGENT_REPORT.md
   ├─ SUBAGENT_REPORT.md
   ├─ REVIEW_REPORT.md
   ├─ БОЛЬШОЙ_БЛОК_ЗАДАЧИ.md
   └─ МАЛЫЙ_БЛОК_ЗАДАЧИ.md
```

## Правило целостности

Если обязательного файла нет, перенос считается неполным.

Если агент добавляет новый файл в шаблон, он обязан обновить этот manifest.

Если задача запрещает менять manifest, агент не добавляет новые обязательные файлы и указывает проблему в отчёте.
