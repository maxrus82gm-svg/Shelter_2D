# Список_правил

Короткий список правил проекта `Shelter_2D`.

Этот документ нужен как промежуточный узел Obsidian-графа: от него идут ссылки на конкретные правила.

## Связи Obsidian

- [[Правила_Git_и_Review]]
- [[Правила_создания_документов]]
- [[Правила_использования_субагентов]]
- [[Настройка_прав_субагентов]]

## Короткий список

1. [[Правила_Git_и_Review]] — Git, GitHub, review, diff, status и запрет Git-операций изменения без прямой команды пользователя.
2. [[Правила_создания_документов]] — создание Markdown-документов, заметок Obsidian, UTF-8 и запрет дублей.
3. [[Правила_использования_субагентов]] — правила контролируемого использования локальных и внешних субагентов.
4. [[Настройка_прав_субагентов]] — центральная ветка настроек, permissions и справочных копий конфигов субагентов.

## Отдельная ветка субагентов

Ветка субагентов подключается отсюда напрямую через `Настройка_прав_субагентов`.

Прямая wiki-связь из этого списка на рабочий документ внешнего субагента не добавляется, чтобы граф оставался последовательным:

```text
Старт -> Задача_агенту -> Список_правил -> Настройка_прав_субагентов
```

Центральный документ настройки прав остаётся здесь:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Настройка_прав_субагентов.md
```

Если задача связана с субагентами, основной Codex читает тематические документы по текстовым путям из `Старт`, `Правила_использования_субагентов` и `Настройка_прав_субагентов`.

## Служебная папка шаблона

`Шаблон_агентской_системы` — это рабочая площадка для сборки будущего единого документа-матрицы правил. Сейчас матрица строится здесь на реальном примере `Shelter_2D`, но не является рабочими правилами проекта.

Основной агент и субагенты не читают и не применяют документы из этой папки в обычных задачах `Shelter_2D`. Если папка будет удалена из проекта, рабочая система правил `Shelter_2D` не должна сломаться.

Если пользователь прямо поставил задачу по созданию, сборке, проверке, изменению, переносу или удалению будущего документа-матрицы, агент работает с этой площадкой только в рамках такой прямой задачи. В остальных случаях ссылки на документы шаблона не добавляются в задачи, prompt для субагентов и список обязательного чтения.

## Рабочий документ внешнего субагента

Рабочий документ внешнего субагента хранится по пути:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_внешнему_субагенту.md
```

Он не подключается отсюда wiki-ссылкой, чтобы внешняя ветка оставалась отдельной и входила в граф через `Задача_агенту` и правила субагентов.

## Главное правило

Перед задачей агент читает `Старт` и `Задача_агенту`.

Если задача требует тематических правил, агент переходит из `Задача_агенту` в этот список и читает только нужные правила.
