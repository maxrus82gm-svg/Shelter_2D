## Что читать перед задачей

Перед началом любой задачи агент читает:

```text
{{START_FILE}}
{{MAIN_TASK_FILE}}
```

Перед нетривиальной задачей агент кратко оценивает, можно ли безопасно использовать внешнего агента или локального субагента для разведки, чтения, черновика, проверки или другой ограниченной подзадачи. Если это снижает расход ресурсов основного агента и не ухудшает качество, контроль, скорость и понятность отчёта, агент предпочитает делегировать такую подзадачу по правилам субагентов.

Если задача требует тематических правил, агент читает только нужные документы из:

```text
{{RULES_FOLDER}}\Список_правил.md
```

`Шаблон_агентской_системы` — это рабочая площадка для сборки будущего единого документа-матрицы правил. Сейчас матрица строится здесь на реальном примере `{{PROJECT_NAME}}`, но не является источником рабочих правил для реальной работы над `{{PROJECT_NAME}}`.

В обычных задачах проекта эту папку не читает и не использует ни основной агент, ни субагенты.

Только если пользователь прямо поставил задачу создать, собрать, перенести, проверить, изменить или удалить будущий документ-матрицу правил, агент читает документы шаблона строго по порядку:

```text
{{DOC_FOLDER}}\Шаблон_агентской_системы\README_как_переносить.md
{{DOC_FOLDER}}\Шаблон_агентской_системы\MANIFEST_структуры.md
{{DOC_FOLDER}}\Шаблон_агентской_системы\00_Паспорт_проекта.md
{{DOC_FOLDER}}\Шаблон_агентской_системы\02_Карта_переменных.md
{{DOC_FOLDER}}\Шаблон_агентской_системы\01_Правило_подгонки_под_новый_проект.md
{{DOC_FOLDER}}\Шаблон_агентской_системы\03_Чеклист_проверки_переноса.md
{{DOC_FOLDER}}\Шаблон_агентской_системы\04_Dry_run_пример_переноса.md
```

Агент не должен начинать работу с частного шаблона, формы отчёта или отдельного правила внутри `Шаблон_агентской_системы`, пока не прочитаны README, manifest, паспорт проекта и карта переменных. Эти документы нельзя применять как рабочие правила текущего проекта, если задача не относится прямо к обслуживанию или переносу шаблона.

## Правило завершения задачи

Если задача успешно выполнена, агент сначала записывает итоговый отчёт в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА` файла:

```text
{{MAIN_TASK_FILE}}
```

Затем агент очищает `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА` до состояния:

```text
Активной задачи нет.
Предыдущая задача выполнена.
Ожидается новая задача от пользователя.
```

Если задача выполнена частично, неуспешно или требует решения пользователя, `БЛОК 1` не очищается, а причина указывается в `PROBLEMS`.

## Правило Git

Git контролирует пользователь.

Политика Git:

```text
{{USER_GIT_POLICY}}
```

Агент не делает commit, push, pull, merge, rebase, reset, clean, checkout, switch или изменение Git-конфига без прямой команды пользователя.

## Правило Obsidian Vault

Настройки `.obsidian` не менять без прямого разрешения пользователя.

## Правило review

Если задача связана с review, проверкой отчёта, Git, GitHub, status, diff, commit, branch или результатом выполненной задачи, ответ начинается с:

```text
СТАТУС КОММИТА
GIT / REPOSITORY CHECK
```

Если commit message не подтверждён, агент пишет `message не подтверждён`. Если push не обнаружен, агент пишет `push не обнаружен`.

## Правило пошагового обучения

Если пользователь просит обучение или разбор, агент объясняет работу пошагово.
