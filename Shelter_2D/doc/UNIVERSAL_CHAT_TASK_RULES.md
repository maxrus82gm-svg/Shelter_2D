# UNIVERSAL_CHAT_TASK_RULES

## Назначение

Этот документ описывает отдельный маршрут для случаев, когда пользователь просит основной Codex подготовить не строгую задачу внешнему субагенту проекта, а универсальный шаблон задания для вставки в другой чат.

Такой шаблон может использоваться для внешнего агента, локального агента в другой среде, обучающего примера, заготовки workflow или разовой задачи, которую пользователь хочет сам отредактировать и отправить.

## Когда использовать

Использовать этот маршрут, если пользователь просит:

- "дай шаблон";
- "сделай шаблон задания в чат";
- "подготовь универсальный prompt";
- "сделай заготовку для внешнего агента";
- "сделай правила немного свободнее";
- "пусть агент сам создаст структуру/папки".

Не использовать этот маршрут, если пользователь просит запустить или провести контролируемую проектную задачу внешнему агенту через рабочий документ `Задача_внешнему_субагенту.md`.

## Отличие от строгого внешнего маршрута

Строгий внешний маршрут:

- привязан к `M:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_внешнему_субагенту.md`;
- требует `БЛОК 1`, `БЛОК 2` и короткого сообщения пользователю;
- подходит для контролируемых задач текущего проекта;
- не даёт агенту свободно расширять структуру.

Универсальный шаблон:

- отдаётся пользователю как текст для чата;
- может быть отредактирован пользователем перед отправкой;
- может разрешать агенту предлагать и создавать папки внутри явно указанной рабочей зоны;
- не обязан использовать `БЛОК 1` и `БЛОК 2`;
- не считается прямым запуском внешнего агента основным Codex.

## Обязательные метки

Если шаблон подготовил основной Codex, в начале шаблона должна быть метка:

```text
РЕЖИМ ШАБЛОНА:
- универсальный шаблон задания в чат

ПОДГОТОВИЛ:
- основной Codex

РАЗДАТЧИК:
- пользователь после проверки и возможной правки шаблона
```

Эта метка означает:

- Codex подготовил заготовку;
- пользователь остаётся финальным раздатчиком задачи;
- пользователь может изменить правила прямо в чате перед отправкой;
- внешний или локальный агент не должен считать такой шаблон строгим проектным маршрутом `Задача_внешнему_субагенту.md`, если это не указано отдельно.

Если пользователь сам пишет задачу без метки `ПОДГОТОВИЛ: основной Codex`, такая задача считается прямым указанием пользователя.

## Гибкость по папкам и файлам

В универсальном шаблоне можно разрешать агенту:

- предложить структуру папок;
- создать новые папки;
- создать новые файлы;
- разложить материалы по подпапкам;
- выбрать имена файлов по смыслу;
- добавить README, заметки, черновики, отчёты или индексные файлы.

Но такая свобода всегда ограничивается явно указанной рабочей зоной:

```text
РАБОЧАЯ ЗОНА:
- <путь или папка, где можно создавать структуру>
```

Если рабочая зона не указана, агент не создаёт файлы сразу. Он сначала предлагает структуру и спрашивает подтверждение.

## Правило русских имён для внешних маршрутов

Файлы и папки, которые создаются специально для внешнего агента, универсального шаблона задания в чат или отчёта агента, по умолчанию должны иметь понятные русские имена.

Использовать:

- `Задача_внешнему_субагенту.md`;
- `Отчёт_агента.md`;
- `Шаблон_задачи_в_чат.md`;
- `Отчёты_агентов`;
- `Черновики`.

Не использовать без прямой просьбы пользователя:

- новые ASCII-имена для пользовательских Markdown-документов;
- имена со спецсимволами;
- пробелы в служебных именах для внешнего агента;
- длинные названия, которые трудно передать в другой терминал.

ASCII-имя допустимо только если есть техническая причина, уже существующий технический файл, внешний инструмент не читает кириллицу или пользователь прямо попросил ASCII. Содержимое отчёта остаётся на русском языке.

## Запреты даже в свободном шаблоне

Даже в универсальном шаблоне агенту запрещено без прямого разрешения пользователя:

- использовать Git;
- менять `.git`;
- менять `.obsidian`;
- менять настройки Codex, OpenCode, VS Code, IDE или shell;
- удалять файлы;
- перемещать существующие папки;
- переписывать существующие документы целиком;
- выходить за рабочую зону;
- работать с секретами, токенами, паролями и приватными ключами.

Если задача требует одного из этих действий, оно должно быть явно перечислено отдельной строкой в разрешениях.

## Правило повреждённой кодировки

Если агент видит в путях, названиях файлов или тексте правил нечитаемую кириллицу, символы замены `�`, наборы `????`, `РџСЂ...`, `Ð...` или другие признаки сломанной кодировки, он не должен угадывать правильные имена файлов и продолжать работу по повреждённым путям.

В этом случае агент обязан:

- остановиться до чтения, создания или изменения файлов по повреждённому пути;
- указать проблему кодировки в `Problems`;
- вернуть статус `частично` или `неуспешно`;
- попросить пользователя или основной Codex дать корректный UTF-8 текст либо путь.

## Обязательный блок маршрута

Универсальный шаблон всё равно должен содержать контроль маршрута:

```text
КОНТРОЛЬ МАРШРУТА АГЕНТОВ:
- Внутренние агенты Codex: `explorer`, `worker`, `default` и другие Codex-native роли. Это не внешние агенты.
- Локальные агенты: запускаются из Codex, но не являются внутренними агентами Codex; пример `simple_local / qwen3-coder:30b / локальный`.
- Внешние агенты: запускаются из других сред, не из Codex; задачу им передаёт пользователь через отдельный чат.
- Этот шаблон является универсальной заготовкой для чата, а не прямым запуском внешнего агента основным Codex.
```

## Минимальный формат универсального шаблона

```text
РЕЖИМ ШАБЛОНА:
- универсальный шаблон задания в чат

ПОДГОТОВИЛ:
- основной Codex

РАЗДАТЧИК:
- пользователь после проверки и возможной правки шаблона

КОНТРОЛЬ МАРШРУТА АГЕНТОВ:
- Внутренние агенты Codex: `explorer`, `worker`, `default` и другие Codex-native роли. Это не внешние агенты.
- Локальные агенты: запускаются из Codex, но не являются внутренними агентами Codex; пример `simple_local / qwen3-coder:30b / локальный`.
- Внешние агенты: запускаются из других сред, не из Codex; задачу им передаёт пользователь через отдельный чат.
- Этот шаблон является универсальной заготовкой для чата, а не прямым запуском внешнего агента основным Codex.

РОЛЬ АГЕНТА:
- <кого пользователь хочет задействовать>

ЦЕЛЬ:
- <что нужно получить>

РАБОЧАЯ ЗОНА:
- <где можно читать/создавать структуру>

ФАЙЛ ОТЧЁТА:
- <путь к Markdown-файлу отчёта, который основной Codex сможет прочитать>
- если путь не указан, создай `AGENT_REPORT.md` в рабочей зоне.

РАЗРЕШЕНО:
- читать материалы в рабочей зоне;
- предлагать структуру папок;
- создавать новые папки и файлы внутри рабочей зоны;
- выбирать имена новых файлов по смыслу;
- записывать итоговый отчёт в файл отчёта.

ЗАПРЕЩЕНО:
- использовать Git;
- менять `.git`, `.obsidian`, настройки IDE, Codex, OpenCode или VS Code;
- удалять или перемещать существующие файлы;
- выходить за рабочую зону;
- работать с секретами.

ПОВРЕЖДЁННАЯ КОДИРОВКА:
- если видишь в путях, названиях файлов или тексте правил нечитаемую кириллицу, символы `�`, `????`, `РџСЂ...`, `Ð...` или другие признаки сломанной кодировки, не угадывай правильные имена файлов и не продолжай работу по повреждённым путям;
- остановись, укажи проблему в `Problems`, верни статус `частично` или `неуспешно` и попроси корректный UTF-8 текст либо путь.

ЕСЛИ РАБОЧАЯ ЗОНА НЕ УКАЗАНА:
- ничего не создавай;
- сначала предложи структуру и запроси подтверждение.

ЯЗЫК ОТЧЁТА:
- итоговый отчёт всегда пиши на русском языке;
- если технические термины, имена файлов, команды или ошибки нужно оставить на английском, оставляй их как есть, но поясняй по-русски.

ЗАВЕРШЕНИЕ РАБОТЫ:
1. Перечитай созданные или изменённые Markdown-файлы и проверь, что они не записаны одной строкой, переносы строк настоящие, кириллица читается нормально.
2. Обязательно запиши итоговый отчёт в `ФАЙЛ ОТЧЁТА`.
3. Если `ФАЙЛ ОТЧЁТА` не указан, создай `AGENT_REPORT.md` в рабочей зоне.
4. Если записать отчёт в файл невозможно, верни полный отчёт в чат и явно укажи причину в `ПРОБЛЕМЫ`.
5. В чат верни короткое резюме, путь к файлу отчёта и финальный маркер.

- <что основному Codex нужно проверить или учесть>

ФОРМАТ ОТВЕТА В ЧАТ:
- краткое резюме на русском;
- путь к файлу отчёта;
- статус;

```
