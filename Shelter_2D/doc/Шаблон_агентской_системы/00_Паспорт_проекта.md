# Паспорт проекта

Заполнить перед переносом шаблонов в рабочий проект.

Паспорт хранит значения, которыми заменяются placeholders вида `{{PROJECT_NAME}}`, `{{PROJECT_ROOT}}`, `{{DOC_FOLDER}}`.

```text
PROJECT_NAME:
PROJECT_ROOT:
OBSIDIAN_VAULT:
DOC_FOLDER:
REPOSITORY_URL:
REPOSITORY_TYPE:
MAIN_TASK_FILE:
EXTERNAL_AGENT_TASK_FILE:
START_FILE:
RULES_FOLDER:
PROJECT_TYPE:
BUILD_SYSTEM:
CODE_FOLDERS:
ASSET_FOLDERS:
CONFIG_FOLDERS:
FORBIDDEN_FOLDERS:
AGENT_CONFIG_PATHS:
SUBAGENT_PROFILES:
MAIN_AGENT_NAME:
LOCAL_AGENT_NAME:
EXTERNAL_AGENT_NAME:
LOCAL_AGENT_ROUTE:
EXTERNAL_AGENT_ROUTE:
SERVICE_FOLDERS:
IDE_SERVICE_FILES:
CURRENT_BASE_COMMIT:
USER_GIT_POLICY:
```

## Как использовать паспорт

1. Сначала заполнить паспорт проекта.
2. Затем подогнать шаблонные документы под проект.
3. Все placeholders заменить значениями из паспорта.
4. Если значение неизвестно, написать `НЕ УКАЗАНО`.
5. Не выдумывать пути, репозитории, ветки, конфиги или профили агентов.

## Минимально обязательные поля

- `PROJECT_NAME`
- `PROJECT_ROOT`
- `OBSIDIAN_VAULT`
- `DOC_FOLDER`
- `REPOSITORY_URL`
- `MAIN_TASK_FILE`
- `EXTERNAL_AGENT_TASK_FILE`
- `START_FILE`
- `USER_GIT_POLICY`
