# README: как переносить агентскую систему

Это универсальный шаблон агентской системы для нового проекта.

Шаблон нужен, чтобы быстро создать набор рабочих документов для агента: стартовую карту, активную задачу, правила Git/review, правила документации, правила субагентов и формы отчётов.

В проекте-источнике эта папка является рабочей площадкой для сборки будущего единого документа-матрицы правил. Сейчас матрица строится на реальном примере проекта-источника, но не является его рабочими правилами и не читается в его обычных задачах.

Template version: 0.2

## Последовательность чтения

Агент читает документы строго в таком порядке:

1. `README_как_переносить.md`
2. `MANIFEST_структуры.md`
3. `00_Паспорт_проекта.md`
4. `02_Карта_переменных.md`
5. `01_Правило_подгонки_под_новый_проект.md`
6. `03_Чеклист_проверки_переноса.md`
7. `04_Dry_run_пример_переноса.md`, если нужно проверить перенос на условном проекте

Нельзя начинать с частного правила или формы отчёта, пока не прочитаны README, манифест и паспорт.

Основные placeholders:

```text
{{PROJECT_NAME}}
{{PROJECT_ROOT}}
{{DOC_FOLDER}}
{{MAIN_TASK_FILE}}
{{REPOSITORY_URL}}
```

## Основной порядок

1. Скопировать папку `Шаблон_агентской_системы` в новый проект.
2. Заполнить `00_Паспорт_проекта.md`.
3. Проверить обязательные поля: `PROJECT_NAME`, `PROJECT_ROOT`, `OBSIDIAN_VAULT`, `DOC_FOLDER`, `REPOSITORY_URL`.
4. Подставить значения из паспорта во все документы из `Шаблоны_документов`.
5. Создать рабочие документы проекта из подготовленных шаблонов.
6. Проверить, что не осталось чужих путей, названий и репозиториев.
7. Записать отчёт о переносе.

## Что внутри

- `00_Паспорт_проекта.md` — единственное место, где фиксируются данные конкретного проекта.
- `01_Правило_подгонки_под_новый_проект.md` — порядок переноса шаблона.
- `02_Карта_переменных.md` — справочник всех placeholders.
- `03_Чеклист_проверки_переноса.md` — контрольный список после переноса.
- `04_Dry_run_пример_переноса.md` — безопасная проверка на условном проекте.
- `MANIFEST_структуры.md` — полный список обязательных файлов шаблона.
- `Шаблоны_документов` — шаблоны рабочих документов агентской системы.
- `Формы_отчётов` — готовые формы отчётов для агента, review и субагентов.

## Важное правило

Шаблоны нельзя использовать как рабочие документы без подстановки переменных.

Если значение неизвестно, агент пишет `НЕ УКАЗАНО`, а не выдумывает.

Основной агент всегда контролирует scope задачи. Git по умолчанию не используется агентами без прямого разрешения пользователя.

Пример: если шаблон переносится в проект `Shelter_2D`, это название можно указывать только после заполнения паспорта проекта.
