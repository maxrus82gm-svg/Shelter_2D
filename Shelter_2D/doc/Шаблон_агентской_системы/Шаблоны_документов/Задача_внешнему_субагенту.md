# Задача_внешнему_субагенту

Рабочий документ для постановки задач внешнему или локальному агенту через чат пользователя.

---

# БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА ВНЕШНЕМУ СУБАГЕНТУ

Активной задачи для внешнего субагента нет.
Предыдущая задача выполнена и проверена основным агентом.
Ожидается новая задача от пользователя.

---

# БЛОК 2 — РАСШИРЕННЫЙ ОТЧЁТ ВНЕШНЕГО СУБАГЕНТА

Отчёта пока нет.

---

# БЛОК 3 — КОРОТКОЕ СООБЩЕНИЕ ДЛЯ ЧАТА ВНЕШНЕМУ СУБАГЕНТУ

```text
Прочитай задачу внешнего субагента:

{{EXTERNAL_AGENT_TASK_FILE}}

Эта задача подготовлена основным агентом.
В начале работы и в отчёте укажи конкретного исполнителя: модель / среда / маршрут.
Не принимай самостоятельные расширения задачи вне БЛОК 1.
Выполни активную задачу из БЛОК 1.
Читай и меняй только файлы, явно перечисленные в scope.
Git не используй, если задача прямо не разрешила.
Не трогай служебные папки и конфиги из паспорта проекта: `{{FORBIDDEN_FOLDERS}}`, `{{AGENT_CONFIG_PATHS}}`.
Сохраняй Markdown как UTF-8 с настоящими переносами строк.
Не очищай БЛОК 1 самостоятельно, если это не разрешено задачей.

По завершении:
1. Верни `USER SUMMARY`.
2. Запиши или верни `AGENT EXTENDED REPORT`.
3. Заверши ответ маркером `РАБОТА ЗАВЕРШЕНА` или `РАБОТА НЕ ЗАВЕРШЕНА`.
```

---

# Проверка перед передачей задачи внешнему агенту

Основной агент перед выдачей короткого сообщения пользователю обязан проверить:

- `БЛОК 1` содержит активную задачу;
- задача относится именно к внешнему агенту;
- scope чтения и правок явно указан;
- куда писать `AGENT EXTENDED REPORT` явно указано;
- короткое сообщение указывает на `{{EXTERNAL_AGENT_TASK_FILE}}`;
- `БЛОК 1` не пустой.

Если `БЛОК 1` пустой, внешний агент не запускается.

Если основной агент обнаружил, что задача внешнего агента ошибочно записана в `{{MAIN_TASK_FILE}}`, он сначала исправляет структуру, а не запускает внешнего агента.

# ПРАВИЛА МАРШРУТА

Основной агент:

- пишет активную задачу в `БЛОК 1`;
- даёт пользователю короткое сообщение из `БЛОК 3`;
- не выполняет задачу сам как внешний агент;
- не пишет `AGENT EXTENDED REPORT` от имени внешнего агента;
- не создаёт и не ждёт внешний чат внутри основного агента, если внешний агент работает через чат пользователя;
- если `БЛОК 1` пустой, не имитирует неуспешный внешний запуск, а сначала записывает задачу в `БЛОК 1` или сообщает пользователю, что внешнему агенту нечего выполнять;
- после выполнения читает короткий отчёт пользователя и расширенный отчёт из `БЛОК 2` или из чата;
- возвращает в основной чат краткую выдержку как `SUBAGENT REPORT`.

Внешний агент:

- читает задачу только из `БЛОК 1`;
- пишет расширенный отчёт в `БЛОК 2`, если это разрешено;
- дублирует `USER SUMMARY` в свой чат для пользователя;
- не очищает `БЛОК 1` самостоятельно.
