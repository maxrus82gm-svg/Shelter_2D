# БЛОК 3 — REVIEW / STATUS

## Большой блок

Большой блок хранится в:

```text
m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_агенту.md
```

Он содержит цель, контекст, scope, разрешённые файлы, запреты, проверки и формат отчёта.

## Малый блок

Малый блок отправляется агенту в чат и дополняет большой блок поздними уточнениями.

Если малый блок противоречит большому, агент останавливается и пишет конфликт в `PROBLEMS`.

Прямое сообщение пользователя имеет приоритет над задачей второго агента.

## Review после отчёта

После отчёта агента основной review начинается с:

```text
СТАТУС КОММИТА
GIT / REPOSITORY CHECK
```

Если GitHub недоступен, писать `message не подтверждён`.

Если изменения локальные и не запушены, писать `push не обнаружен`.

## Контроль размещения активной задачи

- задача основного агента пишется в `m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_агенту.md`;
- задача внешнего агента пишется в `m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_внешнему_субагенту.md`;
- если в основном task-файле найден заголовок `ТЕКУЩАЯ ЗАДАЧА ВНЕШНЕМУ СУБАГЕНТУ`, это ошибка структуры;
- если внешний агент получил короткое сообщение, но `БЛОК 1` внешнего документа пустой, запуск считается неподготовленным;
- основной агент не должен имитировать внешний запуск и не должен писать отчёт от имени внешнего агента.

## КОНТРОЛЬ МАРШРУТА АГЕНТОВ

- Основной агент остаётся координатором, контролирует scope и принимает финальное решение.
- Внутренние Codex-агенты запускаются только средствами Codex и не считаются внешним чатом пользователя.
- Локальный агент используется только при безопасном scope и отдельном разрешении пользователя или активных правил проекта.
- Внешний агент через чат пользователя получает задачу через `m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_внешнему_субагенту.md`.
- Основной агент не имитирует внешний запуск, не пишет отчёт от имени внешнего агента и не смешивает маршруты.
- Git не используется агентами без прямого разрешения пользователя.
