# Паспорт проекта

Заполнить перед переносом шаблонов в рабочий проект.

Паспорт хранит значения, которыми заменяются placeholders вида `{{PROJECT_NAME}}`, `{{PROJECT_ROOT}}`, `{{DOC_FOLDER}}`.

```text
PROJECT_NAME: Shelter_2D
PROJECT_ROOT: m:\GitHub\Shelter_2D\Shelter_2D
OBSIDIAN_VAULT: m:\GitHub\Shelter_2D\Shelter_2D
DOC_FOLDER: m:\GitHub\Shelter_2D\Shelter_2D\doc
REPOSITORY_URL: https://github.com/maxrus82gm-svg/Shelter_2D.git
REPOSITORY_TYPE: GitHub
MAIN_TASK_FILE: m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_агенту.md
EXTERNAL_AGENT_TASK_FILE: m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_внешнему_субагенту.md
START_FILE: m:\GitHub\Shelter_2D\Shelter_2D\doc\Старт.md
RULES_FOLDER: m:\GitHub\Shelter_2D\Shelter_2D\doc
PROJECT_TYPE: C++ 2D game / SFML
BUILD_SYSTEM: Visual Studio 2022 / MSBuild
CODE_FOLDERS: Shelter_2D_Game; Shelter_2D_Game\src
ASSET_FOLDERS: НЕ УКАЗАНО

MAIN_AGENT_NAME: основной агент
LOCAL_AGENT_NAME: локальный агент
EXTERNAL_AGENT_NAME: внешний агент через чат пользователя
LOCAL_AGENT_ROUTE: папка `Локальный агент`, если пользователь отдельно разрешит
EXTERNAL_AGENT_ROUTE: пользователь передаёт малый блок во внешний чат агента

```

## Как использовать паспорт

1. Сначала заполнить паспорт проекта.
2. Затем подогнать шаблонные документы под проект.
3. Все placeholders заменить значениями из паспорта.
4. Если значение неизвестно, написать `НЕ УКАЗАНО`.
5. Не выдумывать пути, репозитории, ветки, конфиги или профили агентов.

## Минимально обязательные поля

- `PROJECT_NAME`
- `PROJECT_ROOT`
- `OBSIDIAN_VAULT`
- `DOC_FOLDER`
- `REPOSITORY_URL`
- `MAIN_TASK_FILE`
- `EXTERNAL_AGENT_TASK_FILE`
- `START_FILE`
- `USER_GIT_POLICY`
