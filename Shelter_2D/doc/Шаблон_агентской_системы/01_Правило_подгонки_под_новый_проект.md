# Правило подгонки под новый проект

Цель: заменить placeholders `{{PROJECT_NAME}}`, `{{PROJECT_ROOT}}`, `{{DOC_FOLDER}}` и остальные переменные значениями из паспорта проекта.

## Порядок переноса

1. Скопировать папку шаблона в новый проект.
2. Заполнить `00_Паспорт_проекта.md`.
3. Проверить, что `PROJECT_NAME`, `PROJECT_ROOT`, `OBSIDIAN_VAULT`, `DOC_FOLDER` и `REPOSITORY_URL` указаны.
4. Подставить значения из паспорта во все шаблонные документы.
5. Проверить, что не осталось старых маркеров другого проекта.
6. Проверить, что нет случайных путей старого проекта.
7. Создать рабочие документы проекта из шаблонов.
8. Не трогать `.git`, `.obsidian` и реальные конфиги без прямого разрешения пользователя.
9. Записать отчёт.

## Правило неопределённости

Если агент не уверен, какой путь или repo использовать, он обязан остановиться и запросить уточнение.

Если поле паспорта не заполнено, агент пишет `НЕ УКАЗАНО`, а не подставляет догадку.

## Проверка после переноса

- Все рабочие документы используют данные текущего проекта.
- В шаблонах не осталось placeholders, которые должны были быть заменены.
- В рабочих документах нет путей чужого проекта.
- Git не использовался, если пользователь отдельно не разрешил Git-проверку.
- Markdown сохранён как UTF-8 с настоящими переносами строк.
