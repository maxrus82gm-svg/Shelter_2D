## Что читать перед задачей

Перед началом любой задачи агент читает:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Старт.md
M:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_агенту.md
```

Если задача требует тематических правил, агент читает только нужные документы из списка:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Список_правил.md
```

Если задача связана с Git, diff, status, review, commit, push, pull, ветками или историей изменений, агент дополнительно читает:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Правила_Git_и_Review.md
```

Если задача связана с созданием или изменением документации, агент дополнительно читает:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Правила_создания_документов.md
```

Если задача предполагает использование субагентов, делегирование, параллельную проверку или выбор между облачным и локальным агентом, агент дополнительно читает:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Правила_использования_субагентов.md
```

Если задача связана с правами субагентов, созданием файлов субагентами, настройкой Codex/OpenCode, permissions, sandbox или тестом записи, агент дополнительно читает:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Настройка_прав_субагентов.md
```

Агент читает только документы, нужные для текущей задачи. Запрещено читать весь Obsidian Vault или весь каталог `doc` подряд без необходимости.

## Правило завершения задачи

Если задача успешно выполнена, агент сначала записывает итоговый отчёт в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА` файла:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_агенту.md
```

Затем агент очищает `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА` до состояния:

```text
Активной задачи нет.
Предыдущая задача выполнена.
Ожидается новая задача от пользователя.
```

Если задача выполнена частично, неуспешно или требует решения пользователя, `БЛОК 1` не очищается, а причина указывается в `PROBLEMS`.

Если задача связана с review, проверкой отчёта, Git, GitHub, status, diff, commit, branch или результатом выполненной задачи, агент читает `Правила_Git_и_Review.md` и начинает ответ с блока `GIT / REPOSITORY CHECK`.

## Правило Obsidian Vault

`Shelter_2D` — это Obsidian Vault проекта.

Заметки проекта создаются и связываются внутри Vault только по прямой задаче пользователя или по правилам документации.

Настройки Obsidian в `.obsidian` менять запрещено без прямой команды пользователя.

## Правило Git

Git полностью контролирует пользователь.

Агент не выполняет Git-операции, которые меняют состояние репозитория: `commit`, `push`, `pull`, `merge`, `rebase`, `reset`, `clean`, `checkout`, `switch`, создание веток, изменение remote, изменение Git-конфига или переписывание истории.

Для Git-задач агент следует документу:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Правила_Git_и_Review.md
```

## Правила C++ / кода

Без прямой задачи пользователя агент не меняет исходный код, настройки сборки, зависимости и структуру проекта.

Если меняются только Markdown-документы, сборка проекта не требуется.

## Правило пошагового обучения

Агент объясняет работу пошагово, если пользователь просит обучение или разбор.

Базовый порядок:

```text
маленькая задача
понятное изменение
проверка
отчёт
следующий шаг
```

Агент не перегружает пользователя лишними документами, командами и теорией без необходимости текущей задачи.
