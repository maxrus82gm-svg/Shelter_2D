Перед началом любой задачи агент всегда читает:


Активная задача находится только в:

```text

```

Если задача связана с Git, GitVerse, diff, status, review, commit, push, pull, ветками или историей изменений, агент дополнительно читает:

```text
M:\GITS_VERSE\VR_Project\VR_Projekt\VR\Doc\Правила_Git_и_Review.md
```

Если задача связана с созданием или изменением документации, агент дополнительно читает:

```text
M:\GITS_VERSE\VR_Project\VR_Projekt\VR\Doc\Правила_создания_документов.md
```

Если задача предполагает использование субагентов, делегирование, параллельную проверку или выбор между облачным и локальным агентом, агент дополнительно читает:

```text
M:\GITS_VERSE\VR_Project\VR_Projekt\VR\Doc\Правила_использования_субагентов.md
```

Краткое универсальное правило: локальный `qwen3-coder:30b / профиль qwen30b-local` используется только как управляемый субагент основного Codex. Пользователь просит Codex поставить ему задачу; Codex формулирует scope, даёт команду для OpenCode-чата, затем сам проверяет отчёт и результат. Субагент работает локально на ПК и Git не использует.

Локальный qwen30b можно держать готовым во время работы через controlled keep-warm: короткий warm-up без чтения и записи файлов. Бесконечные задачи и повторяющиеся псевдо-вызовы инструментов считаются сбоем, а не нормальной работой.

Если задача связана с тем, что локальный или внешний облачный субагент не видит файлы, не может создавать файлы, требует настройки Codex/OpenCode, permissions, sandbox или теста записи, агент дополнительно читает:

```text
M:\GitHub\Shelter_2D\Shelter_2D\doc\Настройка_прав_субагентов.md
```

Этот документ кратко объясняет, где искать реальные конфиги субагентов, какие справочные копии есть в Obsidian и когда нужно просить пользователя перезапустить Codex или OpenCode.

Агент читает только документы, нужные для текущей задачи. Запрещено читать всё Obsidian-хранилище `VR` или весь каталог `Doc` подряд без необходимости.

## Правило Obsidian Vault

`VR` — это Obsidian Vault проекта.

Заметки проекта создаются и связываются внутри `VR` только по прямой задаче пользователя или по правилам документации.

Настройки Obsidian в `.obsidian` менять запрещено без прямой команды пользователя.

## Правило Git

Git полностью контролирует пользователь.

Агент не выполняет Git-операции, которые меняют состояние репозитория: `commit`, `push`, `pull`, `merge`, `rebase`, `reset`, `clean`, `checkout`, `switch`, создание веток, изменение remote, изменение Git-конфига или переписывание истории.

Для Git-задач агент следует документу:

```text
M:\GITS_VERSE\VR_Project\VR_Projekt\VR\Doc\Правила_Git_и_Review.md
```

## Правило чтения GitVerse / SSH для ChatGPT

ChatGPT может видеть только те данные GitVerse и Git, которые пользователь или локальный агент передали в чат: SSH-адрес репозитория, commit hash, branch/ref, вывод Git-команд, diff, отчёты агентов, фрагменты файлов и загруженные файлы.

ChatGPT не имеет прямого SSH-доступа к приватному GitVerse и не имеет доступа к локальному диску пользователя.

Пользователь или локальный агент должны передавать ChatGPT только безопасные данные, нужные для review или текущей задачи.

Запрещено передавать ChatGPT приватные SSH-ключи, пароли, токены, содержимое `~/.ssh/id_*` и секретные конфиги.

Локальный агент может читать Git через SSH только если текущая задача явно разрешает такое чтение.

Если локальный агент читает Git или GitVerse для ChatGPT, он включает в отчёт:

- выполненные Git-команды только на чтение;
- commit hash и branch/ref;
- список прочитанных файлов;
- нужные `diff` / `show` фрагменты;
- подтверждение, что секреты не выводились и не передавались.

ChatGPT обязан честно различать:

- видит SSH-адрес репозитория;
- видит commit hash;
- видит вывод Git-команды;
- видит содержимое файла или diff;
- имеет прямой SSH-доступ.

Если прямого SSH-доступа нет, ChatGPT не должен утверждать, что сам прочитал приватный GitVerse.

## Правило пошагового обучения

Агент объясняет работу пошагово, если пользователь просит обучение или разбор.

Базовый порядок:

```text
маленькая задача
понятное изменение
проверка
отчёт
следующий шаг
```

Агент не перегружает пользователя лишними документами, командами и теориями без необходимости текущей задачи.
