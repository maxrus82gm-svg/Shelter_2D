# Карта переменных

Все переменные записываются в формате `{{VARIABLE_NAME}}`.

| Переменная | Что означает | Пример значения | Где используется | Можно оставить пустой |
|---|---|---|---|---|
| `{{PROJECT_NAME}}` | Название проекта | `Example_Project` | Все основные документы | Нет |
| `{{PROJECT_ROOT}}` | Корень проекта на ПК | `D:\Projects\Example_Project` | Старт, задачи, правила | Нет |
| `{{OBSIDIAN_VAULT}}` | Папка Obsidian Vault | `D:\Projects\Example_Project\Vault` | Документация, Obsidian links | Нет |
| `{{DOC_FOLDER}}` | Папка правил и задач | `D:\Projects\Example_Project\Vault\doc` | Все документы агента | Нет |
| `{{REPOSITORY_URL}}` | URL репозитория | `https://github.com/owner/repository.git` | Git/review правила | Да, если проекта без Git |
| `{{REPOSITORY_TYPE}}` | Тип репозитория | `GitHub`, `GitLab`, `local only` | Git/review правила | Да |
| `{{MAIN_TASK_FILE}}` | Рабочий файл активной задачи | `{{DOC_FOLDER}}\Задача_агенту.md` | Малый блок, старт | Нет |
| `{{EXTERNAL_AGENT_TASK_FILE}}` | Рабочий файл задачи внешнему агенту | Пример: `{{DOC_FOLDER}}\Задача_внешнему_субагенту.md` | Правила субагентов, задачи внешнему агенту | Да, если внешние агенты не используются |
| `{{START_FILE}}` | Стартовый документ | `{{DOC_FOLDER}}\Старт.md` | Старт, карта чтения | Нет |
| `{{RULES_FOLDER}}` | Папка правил | `{{DOC_FOLDER}}` | Все правила | Нет |
| `{{PROJECT_TYPE}}` | Тип проекта | `C++ game`, `web app`, `docs only` | Старт, правила кода | Да |
| `{{BUILD_SYSTEM}}` | Система сборки | `CMake`, `npm`, `НЕ УКАЗАНО` | Старт, проверки | Да |
| `{{CODE_FOLDERS}}` | Папки исходного кода | `src; include` | Правила кода | Да |
| `{{ASSET_FOLDERS}}` | Папки ассетов | `assets; media` | Правила проекта | Да |
| `{{CONFIG_FOLDERS}}` | Папки конфигов проекта | `config; tools` | Запреты и проверки | Да |
| `{{FORBIDDEN_FOLDERS}}` | Папки, которые нельзя менять без разрешения | Пример: `.git; .obsidian; vendor` | Запреты | Нет |
| `{{AGENT_CONFIG_PATHS}}` | Пути реальных конфигов агентов | `C:\Users\User\.codex; C:\Users\User\.config\opencode` | Настройка субагентов | Да |
| `{{SUBAGENT_PROFILES}}` | Профили субагентов | `simple_local; qwen-local; cloud-agent` | Правила субагентов | Да |
| `{{MAIN_AGENT_NAME}}` | Название основного агента в проекте | Пример: `Main AI agent` | Правила субагентов, отчёты | Нет |
| `{{LOCAL_AGENT_NAME}}` | Название локального агента или локального профиля | Пример: `local-helper` | Правила субагентов, отчёты | Да |
| `{{EXTERNAL_AGENT_NAME}}` | Название внешнего агента или внешнего профиля | Пример: `external-review-agent` | Правила внешних агентов, отчёты | Да |
| `{{LOCAL_AGENT_ROUTE}}` | Как основной агент ставит задачу локальному агенту | Пример: `локальная вкладка субагента в IDE` | Правила субагентов | Да |
| `{{EXTERNAL_AGENT_ROUTE}}` | Как пользователь передаёт задачу внешнему агенту | Пример: `чат внешнего агента в отдельной IDE` | Правила внешних агентов | Да |
| `{{SERVICE_FOLDERS}}` | Служебные папки проекта | Пример: `.git; .idea; .cache` | Git/review, запреты, unexpected changes | Да |
| `{{IDE_SERVICE_FILES}}` | Служебные файлы IDE, workspace, session и cache | Пример: `.vscode/*; workspace.json; *.session` | Git/review, unexpected changes | Да |
| `{{CURRENT_BASE_COMMIT}}` | Базовый commit для сверки | `abc1234` или `НЕ УКАЗАНО` | Review, Git правила | Да |
| `{{USER_GIT_POLICY}}` | Политика пользователя по Git | `Git не использовать без прямой команды` | Git/review правила | Нет |

## Правило заполнения

Если значение неизвестно, писать `НЕ УКАЗАНО`.

Если проект не использует Git, `REPOSITORY_TYPE` можно указать как `local only`, а `REPOSITORY_URL` как `НЕ УКАЗАНО`.
