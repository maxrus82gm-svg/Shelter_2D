# Правила создания документов

## Назначение

Документ описывает правила создания и обновления Markdown-документов в проекте `Shelter_2D`.

## Базовые правила

- Документы создаются только по задаче пользователя или активной задаче в `m:\GitHub\Shelter_2D\Shelter_2D\doc\Задача_агенту.md`.
- Документы сохраняются в Markdown.
- Кодировка: UTF-8.
- Русский текст должен читаться нормально.
- Использовать настоящие переносы строк.
- Не записывать файл одной строкой.
- Не вставлять текстовую последовательность "обратная косая черта + n" вместо переносов.
- Не экранировать Windows-пути двойными слэшами.
- Не менять `.obsidian` без разрешения.
- Не менять реальные конфиги агентов без прямой задачи.

## Правила папки шаблона агентской системы

`Шаблоны_документов` — это рабочая площадка для сборки будущего единого документа-матрицы правил. Сейчас матрица строится здесь на реальном примере `Shelter_2D`, но не используется как рабочий набор правил текущего проекта `Shelter_2D`.

Папка универсального шаблона агентской системы находится здесь:

```text
m:\GitHub\Shelter_2D\Shelter_2D\doc\Шаблоны_документов
```

Если пользователь прямо поставил задачу создать, собрать, перенести, проверить, изменить или удалить будущий документ-матрицу правил, агент обязан сначала читать документы шаблона в порядке:

```text
README_как_переносить.md
MANIFEST_структуры.md
00_Паспорт_проекта.md
02_Карта_переменных.md
01_Правило_подгонки_под_новый_проект.md
03_Чеклист_проверки_переноса.md
04_Dry_run_пример_переноса.md
```

Изменения в шаблоне и изменения в рабочих правилах проекта не должны расходиться по смыслу. Если новое универсальное правило добавляется в шаблон и оно влияет на текущую работу проекта, агент добавляет отражение этого правила в рабочие документы `m:\GitHub\Shelter_2D\Shelter_2D\doc`.

Если новое правило добавляется в рабочие документы и должно переноситься в будущие проекты, агент добавляет его отражение в шаблонные документы с placeholders вида `Shelter_2D`, `m:\GitHub\Shelter_2D\Shelter_2D`, `m:\GitHub\Shelter_2D\Shelter_2D\doc`.

В обычных задачах текущего проекта агент не берёт инструкции из `Шаблоны_документов`, не подменяет ими рабочие правила и не передаёт их выбранному исполнителю как источник правил.

Если в папку шаблона добавлен новый обязательный файл, агент обновляет `MANIFEST_структуры.md` и при необходимости `README_как_переносить.md`, чтобы следующий агент не пропустил этот файл.

## Obsidian links

Если проект использует Obsidian, новые документы связываются через Obsidian links, когда это нужно проекту:

```text
Название_документа
```

## Проверка перед завершением

- Документ находится в разрешённом scope.
- Нет дубля уже существующего документа.
- Markdown читается нормально.
- UTF-8 сохранён.
- Нет текстовой последовательности "обратная косая черта + n" вместо переносов.
- Нет случайных путей чужого проекта.
