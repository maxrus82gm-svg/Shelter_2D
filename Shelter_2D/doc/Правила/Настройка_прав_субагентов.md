# Настройка прав субагентов

## Назначение

Документ описывает, где проверять права внешних и локальных агентов проекта `Shelter_2D`.

## Карта настроек

Реальные настройки агентов:

```text
НЕ УКАЗАНО
```

Профили субагентов:

```text
НЕ УКАЗАНО
```

## Главное правило

Модель может уметь писать файлы, но право на запись выдаёт среда запуска: Codex, OpenCode, VS Code, sandbox, permissions и доступные tools.

## Что проверять

- Есть ли у агента локальный доступ к `m:\GitHub\Shelter_2D\Shelter_2D`.
- Может ли агент читать `m:\GitHub\Shelter_2D\Shelter_2D\doc`.
- Разрешена ли запись в явно указанный scope.
- Не требует ли среда подтверждения пользователя.
- Не запущен ли агент в read-only режиме.
- Запрещён ли Git текущей задачей.

## Запреты

- Не менять реальные конфиги без прямой задачи пользователя.
- Не расширять permissions без понимания риска.
- Не использовать Git для проверки permissions.

## Отчёт

Если агент не может читать или писать файлы, в отчёте указать:

```text
Agent:
Route:
Missing permission:
Unavailable tool:
Requested path:
Result:
```
