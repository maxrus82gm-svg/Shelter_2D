# Правила использования субагентов

Проект:

```text
{{PROJECT_NAME}}
```

Рабочая директория:

```text
{{PROJECT_ROOT}}
```

Рабочий документ внешнего субагента:

```text
{{DOC_FOLDER}}\Задача_внешнему_субагенту.md
```

## Главное правило

Основной агент принимает финальное решение, контролирует scope и проверяет результат.

Субагент не расширяет scope и выполняет только задачу, подготовленную основным агентом.

## Внешний или локальный агент через чат пользователя

Внешний или локальный агент может работать через чат пользователя в отдельной среде.

Порядок:

1. Основной агент пишет задачу и scope.
2. Пользователь передаёт задачу в чат субагента.
3. Субагент выполняет только указанную задачу.
4. Субагент возвращает `USER SUMMARY`.
5. Субагент пишет или возвращает `AGENT EXTENDED REPORT`.
6. Основной агент проверяет результат и принимает решение.

## Запреты

- Субагент не использует Git без прямого разрешения.
- Субагент не очищает `БЛОК 1`, если это не разрешено задачей.
- Субагент не меняет файлы вне scope.
- Субагент не меняет `.git`, `.obsidian` и реальные конфиги без разрешения.

## Финальный маркер

Субагент завершает работу маркером:

```text
РАБОТА ЗАВЕРШЕНА
Итог: успешно
Контроль возвращён основному Codex.
```

или:

```text
РАБОТА НЕ ЗАВЕРШЕНА
Итог: частично / неуспешно
Контроль возвращён основному Codex.
```

Без маркера отчёт считается неполным.

## Форматность Markdown

Субагент сохраняет Markdown как UTF-8 с настоящими переносами строк, не вставляет буквальные `\n` и не экранирует Windows-пути двойными слэшами.
