# Правила Git и Review

## Назначение

Документ описывает безопасную работу с Git, GitHub, diff, status, review и историей изменений в проекте `Shelter_2D`.

Документ читается только тогда, когда задача связана с Git, GitHub, commit, push, pull, ветками, diff, status, review, историей изменений или проверкой результата.

## Главное правило Git

Git контролирует только пользователь.

Агенту запрещено без прямой команды пользователя выполнять действия, которые меняют состояние репозитория, рабочей копии, веток, remote, Git-конфига или истории.

Запрещены:

```bash
git add
git commit
git push
git pull
git merge
git rebase
git reset
git clean
git checkout
git switch
git branch
git tag
git remote
git config
```

Агент не создаёт commit.  
Агент не делает push.  
Агент не делает pull.  
Агент не создаёт ветки.  
Агент не переключает ветки.  
Агент не меняет remote.  
Агент не меняет Git-конфиг.  
Агент не переписывает историю Git.

## Что агенту разрешено

Агент может читать состояние Git только при необходимости текущей задачи и только если задача это разрешает.

Разрешены только безопасные команды просмотра:

```bash
git status
git status --short
git diff
git diff --stat
git diff --name-only
git log
git log --oneline
git show <commit>
git ls-files --modified
git ls-files --others --exclude-standard
```

Эти команды используются только для чтения. Если агент использовал Git-команды, он перечисляет их в отчёте.

Если текущая задача прямо запрещает Git, не выполнять даже команды чтения.

## Статус чтения GitHub

Если агент читает репозиторий GitHub или данные, связанные с GitHub, он явно указывает статус чтения:

```text
Repository: https://github.com/maxrus82gm-svg/Shelter_2D.git
Branch/ref:
Commit SHA:
Commit message:
Files read:
Проверка:
```

Если пользователь дал конкретный commit, агент указывает, что чтение выполнено по этому commit.

Если commit или ветку подтвердить нельзя, агент пишет причину.

## Формат REVIEW

Review — это проверка результата, diff, commit или состояния репозитория.

Формат:

```text
REVIEW

Repository: https://github.com/maxrus82gm-svg/Shelter_2D.git
Branch/ref:
Commit SHA:
Commit message:
Files read:

FINDINGS:
- найденные проблемы или "Проблем не найдено"

RISKS:
- риски, если есть

NEXT:
- следующий безопасный шаг
```

Если проблема найдена на review, агент не исправляет её сам без прямой команды пользователя. Агент описывает проблему и предлагает следующий безопасный шаг.

## Правило локальных изменений

Агент не утверждает, что видел локальные изменения, если он их реально не проверял.

Если проверялся локальный путь, указывается:

```text
Проверен локальный путь:
M:\GitHub\Shelter_2D
```

Если Git-команды чтения не выполнялись, отчёт должен прямо сказать:

```text
Git-команды чтения не выполнялись.
```

## Обязательное начало review после выполненной задачи

Любой review после выполненной задачи начинается с блока:

```text
GIT / REPOSITORY CHECK
```

Этот блок обязателен даже если Git не использовался.

Если Git не проверялся, нельзя писать так, будто Git-состояние известно.

Если пользователь дал только текстовый отчёт агента, нужно прямо указать:

```text
Git-состояние по репозиторию не подтверждено, потому что нет вывода git status / branch / diff.
Оценка выполнена только по текстовому отчёту агента.
```

Обязательный формат блока:

```text
GIT / REPOSITORY CHECK

Repository:
- ссылка на репозиторий, если известна
- если не известна: не указано

Branch:
- текущая ветка, если проверялась
- если не проверялась: не проверялась

Status:
- есть ли незакоммиченные изменения, если проверялось
- если не проверялось: не проверялось

Changed files:
- какие файлы изменены, если проверялось
- если не проверялось: не проверялось
- если есть только отчёт агента: указать "по отчёту агента"

Diff summary:
- кратко что изменилось, если diff проверялся
- если diff не проверялся: не проверялся

Commit:
- текущий commit, если проверялся
- commit не создавался агентом

Push:
- не выполнялся агентом

Pull:
- не выполнялся агентом

GitHub check:
- проверялся / не проверялся
```

Если Git был запрещён задачей, использовать такой вариант:

```text
GIT / REPOSITORY CHECK

Repository:
- указано в задаче или отчёте, если есть

Branch:
- не проверялась, потому что Git был запрещён задачей

Status:
- не проверялся, потому что Git был запрещён задачей

Changed files:
- по отчёту агента: перечислить заявленные изменённые файлы
- через Git не проверялось

Diff summary:
- через Git не проверялся

Commit:
- не создавался агентом

Push:
- не выполнялся агентом

Pull:
- не выполнялся агентом

GitHub check:
- не проверялся
```

## Обязательная проверка последнего commit после отчёта агента

Если пользователь принёс отчёт Codex, OpenCode, главного агента, локального агента, облачного агента или субагента о выполненной задаче, основной агент / ChatGPT обязан сначала попытаться проверить GitHub и получить commit message последнего commit на актуальной ветке проекта.

Обязательный формат начала review:

```text
СТАТУС КОММИТА: "<commit message>"
Repository: owner/repo
Branch: main
Commit: <short sha>
Проверка: изменения найдены / push не обнаружен / требуется уточнение
```

Важно:

- этот блок должен идти самым первым;
- нельзя сначала писать "отлично", "принято", "выглядит нормально";
- нельзя принимать отчёт только по словам агента;
- сначала проверяется последний commit;
- потом пишется `GIT / REPOSITORY CHECK`;
- только потом пишется `REVIEW STATUS`.

Если commit message доступен, агент указывает именно его.

Если commit message короткий или неинформативный, например:

```text
+
01
02
fix
test
```

агент всё равно указывает его как есть.

Запрещено:

- улучшать commit message;
- переписывать commit message;
- додумывать commit message;
- заменять короткий message на красивое описание;
- скрывать неинформативный message.

Правильный пример:

```text
СТАТУС КОММИТА: "+"
Repository: maxrus82gm-svg/Shelter_2D
Branch: main
Commit: a1b2c3d
Проверка: изменения найдены
```

Если пользователь принёс отчёт о выполненной задаче, но в GitHub последний commit не содержит заявленных изменений, агент пишет:

```text
СТАТУС КОММИТА: push не обнаружен
Repository: owner/repo
Branch: main
Commit: <short sha последнего доступного commit>
Проверка: изменения не подтверждены через GitHub
```

После этого агент должен пояснить:

```text
Изменения могут быть только локально на ПК пользователя.
Через GitHub они пока не подтверждены.
Для полной проверки нужен push, commit SHA, git status, git diff или содержимое изменённых файлов.
```

Если commit message не удалось подтвердить, агент пишет:

```text
СТАТУС КОММИТА: message не подтверждён
Repository: owner/repo
Branch/ref: <branch/ref или не подтверждён>
Commit: <short sha или не подтверждён>
Проверка: требуется уточнение
```

Если не подтверждён даже commit SHA:

```text
СТАТУС КОММИТА: message не подтверждён
Repository: owner/repo
Branch/ref: не подтверждён
Commit: не подтверждён
Проверка: требуется уточнение
```

Блок `СТАТУС КОММИТА` не заменяет `GIT / REPOSITORY CHECK`.

Правильный порядок review:

```text
СТАТУС КОММИТА
GIT / REPOSITORY CHECK
REVIEW STATUS
FINDINGS
PROBLEMS
NEXT
```

Пример:

```text
СТАТУС КОММИТА: "02"
Repository: maxrus82gm-svg/Shelter_2D
Branch: main
Commit: a1b2c3d
Проверка: изменения найдены

GIT / REPOSITORY CHECK

Repository:
- проверен GitHub: maxrus82gm-svg/Shelter_2D

Branch:
- проверена ветка: main

Status:
- локальный status не проверялся
- GitHub-содержимое проверено по последнему commit

Changed files:
- проверены по GitHub / commit / отчёту агента

Diff summary:
- кратко описать, что изменилось

Commit:
- a1b2c3d
- message: "02"

Push:
- не выполнялся основным агентом
- наличие изменений в GitHub подтверждает, что push уже был выполнен пользователем или другим агентом

Pull:
- не выполнялся основным агентом

GitHub check:
- проверялся
```

GitHub показывает только то, что уже попало в репозиторий.

Основной агент не должен утверждать, что проверил локальные незакоммиченные изменения, если нет:

```text
git status
git diff
git log -1
содержимого изменённых файлов
```

Если изменения есть только локально и не запушены, GitHub-проверка должна завершиться статусом:

```text
Проверка: push не обнаружен
```

## Правило отчёта для Git-задачи

Если задача затрагивала Git, агент указывает:

```text
Branch/ref:
- текущая ветка или ref, если проверялись

Status:
- состояние, если проверялось

Changed files:
- изменённые файлы, если проверялись

Diff summary:
- краткое описание diff, если проверялся

Commit:
- не создавался агентом

Push:
- не выполнялся агентом

GitHub check:
- проверялся или нет
```

Если Git/review-задача успешно выполнена, агент сначала записывает итоговый отчёт в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА` рабочего документа `Задача_агенту.md`, а затем очищает `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА`.

Если задача выполнена частично, неуспешно или требует решения пользователя, `БЛОК 1` не очищается, а причина указывается в `PROBLEMS`.

## Главное правило

Git используется агентом только для чтения и анализа, если это нужно задаче и прямо разрешено.

Commit, push, pull, branch, merge, rebase, reset, checkout, switch, clean, изменение remote, изменение Git-конфига и переписывание истории выполняет только пользователь.
